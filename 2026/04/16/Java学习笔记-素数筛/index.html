Java 学习笔记：高效查找素数的埃氏筛法

在算法面试和日常编程中，查找一定范围内的所有素数是一个经典问题。相比于传统的逐个判断法，埃拉托斯特尼筛法（Sieve of Eratosthenes） 效率极高。本文将总结其核心思想并提供 Java 实现。

1. 核心思想

埃氏筛的基本逻辑是：从最小的素数开始，将其所有的倍数（合数）标记出来并剔除。

具体步骤如下：

  1. 创建一个布尔数组，初始化 2 以上的所有数字为“待定素数”。
  2. 从最小的素数 2 开始，将其倍数（如 4, 6, 8, 10…）全部标记为“非素数”。
  3. 移动到下一个未被标记的数字（即 3），重复上述剔除过程。
  4. 优化：只需要筛选到 $\sqrt{n}$ 即可，因为更大的合数必然在之前已被较小的因数筛选过。

3. 复杂度分析

  • 时间复杂度：$O(n \log \log n)$。这接近于线性时间，比逐个判断的 $O(n\sqrt{n})$ 要快得多。
  • 空间复杂度：$O(n)$。需要一个长度为 $n+1$ 的布尔数组来存储标记状态。

4. 总结与反思

埃氏筛是“空间换时间”思想的典型应用。通过一次性标记，避免了对每个数字进行大量的除法运算。在处理百万级别范围内的素数查找时，这种算法的优势会非常明显。
