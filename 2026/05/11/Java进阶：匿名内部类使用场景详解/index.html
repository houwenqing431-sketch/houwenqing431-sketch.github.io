Java 进阶：匿名内部类（Anonymous Inner Class）的使用场景详解

在 Java 开发中，匿名内部类是一种非常实用的特性。它允许我们在声明一个类的同时将其实例化。当我们只需要某个类的一个实例，且该类不需要在其他地方重复使用时，匿名内部类是减少冗余代码、提高开发效率的神器。

本文将深入探讨匿名内部类的核心使用场景，并对比其在现代 Java（Lambda）背景下的地位。

一、 核心概念

匿名内部类本质上是一个没有名字的局部内部类。它隐含地继承了一个父类或者实现了一个接口。

二、 三大典型使用场景

1. 临时开启多线程（实现 Runnable）

在不需要定义复杂的线程逻辑时，匿名内部类可以让我们在 new Thread 的同时直接写好运行逻辑，而不需要单独写一个实现类。

2. 集合自定义排序（实现 Comparator）

在对 List 进行排序时，我们往往需要根据特定的业务规则（如按字符串长度、按对象某个属性值）进行比较。这时候传入一个匿名的 Comparator 是最常见的做法。

3. 事件监听与回调机制（Event Listeners）

在桌面应用开发（Swing/AWT）或者 Android 开发中，处理按钮点击、鼠标移动等事件时，大量使用匿名内部类来实现回调。

三、 匿名内部类 vs Lambda 表达式

在 Java 8 之后，Lambda 表达式的出现让代码更加简洁。对于只有一个抽象方法的接口（函数式接口），我们现在通常写成 Lambda 形式：

但是，在以下场景中，匿名内部类依然不可替代：

  1. 接口中有多个抽象方法：Lambda 只能用于单方法的函数式接口。
  2. 需要重写抽象类的方法：Lambda 只能用于接口，不能用于抽象类。
  3. 需要在匿名类中定义自己的成员变量或多个方法。

四、 总结

匿名内部类是 Java 语言灵活性的一种体现，它通过“即用即毁”的机制，极大地简化了简单回调和一次性任务的实现。虽然 Lambda 在很多地方取代了它，但理解匿名内部类依然是掌握 Java 内部类体系、多态以及作用域规则（如对局部变量的访问限制）的重要基础。
