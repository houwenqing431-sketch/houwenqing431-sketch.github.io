Java I/O 实战：高效文件复制的“万能”与“专用”流

在 Java 开发与算法竞赛中，文件的读写与复制是极其高频的操作。今天的“Homework to Blog”任务聚焦于 Java 的 I/O 流。

无论是复制简单的文本，还是复制体积庞大的音视频文件，缓冲流（Buffered Stream） 都是提升程序运行效率的核心利器。本文将梳理“文本文件”与“任意文件”复制的最佳实践。

一、 为什么要用缓冲流？

在基础的 FileInputStream 或 FileReader 中，程序每次读写通常是直接与硬盘进行交互，速度极慢。

而缓冲流（如 BufferedInputStream 或 BufferedReader）在内存中开辟了一个默认 8KB 的缓冲区。这就好比搬砖：基础流是一块一块地搬，而缓冲流是给你一辆推车，装满一车再批量运送，极大地减少了磁盘 IO 的次数，性能呈指数级飞跃。

二、 场景一：文本文件复制（最常用专用流）

如果明确知道你要复制的是纯文本文件（如 .txt, .java, .md, .csv），字符缓冲流是绝对的首选。

1. 核心类

  • BufferedReader：字符缓冲输入流。它提供了一个极为好用的 readLine() 方法，可以一次读取一整行文本。
  • BufferedWriter：字符缓冲输出流。它提供了一个 newLine() 方法，能根据不同操作系统自动生成对应的换行符。

三、 场景二：任意文件复制（万能通用流）

如果你要复制的是图片 (.png)、视频 (.mp4)、可执行文件 (.exe) 甚至是压缩包，绝对不能用字符流（字符编码转换会破坏原本的二进制数据，导致图片打不开或视频黑屏）。

这时候必须请出“万能复制”工具——字节缓冲流。因为计算机底层存储的所有文件，本质上都是由字节（Byte）组成的。

1. 核心类

  • BufferedInputStream：字节缓冲输入流。
  • BufferedOutputStream：字节缓冲输出流。

四、 总结与避坑指南

  1. 选型口诀：处理纯文本，首选字符缓冲流（按行操作太香了）；处理非文本或未知类型文件，无脑用字节缓冲流（万能且绝对安全）。
